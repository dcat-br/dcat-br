Campos Mínimos por Nível

Esta seção apresenta os campos mínimos obrigatórios para cada nível de qualidade (Bronze, Prata e Ouro) conforme definido na Metodologia de Qualidade em Metadados (MQM).

Legenda FAIR+C: E = Encontrabilidade, A = Acessibilidade, I = Interoperabilidade, R = Reutilizabilidade, C = Contextualidade. Os valores entre parênteses indicam a pontuação máxima que o campo pode contribuir para a dimensão.

Bronze Campos Mínimos - Nível Bronze

Campos do Conjunto de Dados

Campo Descrição FAIR+C / Pontuação
dcterms:title Título do conjunto de dados é obrigatório. -
dcterms:description Descrição do conjunto de dados é obrigatório. -
dcat:keyword Deve conter no minimo uma palavra-chave. E (20)
dcterms:publisher Área técnica responsável pelo dado é obrigatório. R (10)
dcat:contactPoint E-mail da área técnica responsável é obrigatório. R (20)
dcterms:accrualPeriodicity Periodicidade de atualização é obrigatória. -
dcterms:license Licença de uso é obrigatória. R (20+10)
dcterms:accessRights Observância Legal é obrigatória. R (10+5)
dcat:theme Tema é obrigatório. E (30+10)

Campos FAIR+C Não Incluídos nos Campos Mínimos

Esta tabela apresenta os campos avaliados nas métricas FAIR+C que não fazem parte dos campos mínimos obrigatórios, mas que contribuem para a pontuação total da qualidade dos metadados.

Campo Descrição FAIR+C / Pontuação
dct:temporal Verifica se metadados temporais foram definidos, permitindo a filtragem de dados por período. E (20)
dcat:downloadURL A URL de transferência (dcat:downloadURL) deve ser um link direto para o download dos dados. A (50)
dcat:mediaType Especifica o tipo de mídia da distribuição (dcat:mediaType). Verifica se a propriedade está definida. I (10+10)
dct:issued A data em que o conjunto de dados ou a distribuição foi divulgado (dct:issued). C (5)
dct:modified A data em que o conjunto de dados ou a distribuição foi alterado pela última vez (dct:modified). C (5)

Nota: A validação SHACL (Conformidade com o DCAT-BR) também contribui com 30 pontos para a dimensão de Interoperabilidade, mas não é um campo específico, e sim uma validação geral dos metadados.
